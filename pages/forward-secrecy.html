Perfect Forward Secrecy

What happens when a long-term private key is leaked after the fact? With static keys, every past session is retroactively compromised. Perfect Forward Secrecy uses ephemeral (one-time) keys so that past sessions remain secure even if the long-term key is later stolen.

Scenario: Static Keys (No PFS)
Alice and Bob reuse the same long-term DH key pair for every session. An adversary records all encrypted traffic. Later, Alice's private key is stolen. The adversary can now decrypt every recorded session retroactively.

Step-by-Step Breakdown

1

Multiple Sessions Established

Alice and Bob communicate over three sessions. Each session produces an encrypted ciphertext that an adversary records passively.

2

Key Compromise Event

At some point in the future, Alice's private key is stolen — either by a breach, legal compulsion, or cryptanalytic breakthrough.

3

Consequence Depends on Key Design

Without PFS: the stolen long-term key lets the attacker recompute all past session keys from the recorded public key exchanges → all sessions decrypted. With PFS: ephemeral keys were discarded after each session; the long-term key cannot reconstruct them → past sessions remain secret.
